Job DAG (each `if` strict to `main` push or `workflow_dispatch`):

| Job              | Trigger             | Steps (key)                                                            |
| ---------------- | ------------------- | ---------------------------------------------------------------------- |
| `test-fast`      | every PR/push       | checkout · setup-python · setup-uv · `uv sync --dev` · `make check`   |
| `test-full`      | main push / dispatch| + apt install shellcheck · `make verify`                              |
| `build`          | main push / dispatch| `docker build -t $IMG:$SHA` · tag latest · save to `image.tar.gz` artifact |
| `push-ecr`       | main push / dispatch| download artifact · validate vars · OIDC login · ecr-login · tag+push `$SHA` and `latest` |
| `deploy`         | main push / dispatch| validate vars · OIDC · `apprunner start-deployment` (capture op_id) · poll `list-operations` until SUCCEEDED/SUCCESS (max 60×10s) · `curl /health` (max 30×10s) · job summary |

Polling decision tree in `deploy`:
```bash
case "$status" in
  SUCCEEDED|SUCCESS)                                                  → break
  FAILED|ROLLBACK_FAILED|CANCELLED|CREATE_FAILED|DELETE_FAILED)       → exit 1
  *)                                                                  → sleep 10
esac
```

Required GitHub repository **variables** (no long-lived secrets):
`AWS_ROLE_TO_ASSUME`, `AWS_REGION`, `ECR_REPOSITORY`, `APPRUNNER_SERVICE_ARN`.

### `apprunner-domain-setup.sh` — associate custom domain + Route 53 records
```bash
# env+flags: --domain, --service-name, --service-arn, --region, --hosted-zone-id,
#            --apprunner-hosted-zone-id, --no-www, --skip-route53,
#            --skip-hosted-zone-create, --help
# Flow:
#   1. Resolve SERVICE_ARN via lookup_service_arn (or use --service-arn)
#   2. SERVICE_URL via describe_service_url
#   3. Resolve / create HOSTED_ZONE_ID (lookup_hosted_zone_id → ensure_hosted_zone)
#   4. aws apprunner associate-custom-domain (+ --enable-www-subdomain unless --no-www)

#        - CertificateValidationRecords as CNAMEs (TTL 300)
#        - apex A ALIAS → SERVICE_URL via APPRUNNER_HOSTED_ZONE_ID
#        - www CNAME → SERVICE_URL (if --no-www not set)
#   6. aws route53 change-resource-record-sets --change-batch file://...
#   7. Print nameservers + current domain status
```

## Conventions / Invariants

- **Linux-first.** Bash + Makefile + WSL on Windows. Do not extend `deployment/windows/legacy/`.
- **Settings are frozen.** `Settings` is `@dataclass(frozen=True)` with `__post_init__` normalizing paths and stripping trailing `/` from `site_url`.
- **Caching.** `get_settings` is `lru_cache(maxsize=1)`; `load_json_file` is `lru_cache(maxsize=None)`. Tests reset both via `clear_settings_cache()` / `clear_content_cache()` in an autouse fixture.
- **Coverage gate is hard:** `--cov-fail-under=90` for `rickarko_portfolio` (enforced by `make test` and `make verify`).
- **Sitemap order:** `home, experience, projects, blog, contact` — encoded in `build_sitemap_pages` and asserted in tests.
- **Schema graph only on home.** `build_page("home")` adds `schema_graph`; other pages must not.
- **WSGI dual-export:** `wsgi.app` and `wsgi.application` must remain pointing at the same Flask instance (regression test enforces).
- **OIDC, not access keys:** GitHub Actions assumes an IAM role via OIDC and is restricted to `refs/heads/main` in the trust policy.
- **App Runner deploy waits twice:** for the service operation to reach SUCCEEDED, then for `https://rickarko.com/health` to respond 2xx.
